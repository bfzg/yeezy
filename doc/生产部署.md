# 生产部署文档

本文档用于把 YEZI Shop 部署到一台 Ubuntu VPS，并绑定独立域名。

当前项目使用 SQLite 和本地上传目录，因此推荐先用 VPS 部署。不要直接部署到无持久磁盘的平台，否则 `data/yezi.db` 和 `public/uploads/` 可能丢失。

## 1. 部署目标

建议生产结构：

```text
域名 -> Nginx -> Next.js app:40322 -> SQLite data/yezi.db
```

最低服务器配置：

```text
Ubuntu 22.04 / 24.04
1 核 CPU
1-2GB RAM
20GB 磁盘
```

## 2. 域名解析

在域名服务商后台添加 DNS 记录：

```text
类型    主机名    值
A       @         服务器公网 IP
A       www       服务器公网 IP
```

本文后续用 `dropmini.com` 举例。实际部署时替换成你的真实域名。

如果服务器上已经有其他站点，例如 `etedrop.com`，可以继续共用同一个公网 IP。Nginx 会根据访问的域名分流：

```text
etedrop.com     -> 43.153.143.37 -> Nginx -> etedrop 服务
dropmini.com    -> 43.153.143.37 -> Nginx -> dropmini 服务
www.dropmini.com -> 43.153.143.37 -> Nginx -> dropmini 服务
```

也就是说，`dropmini.com` 和 `www.dropmini.com` 的 A 记录直接指向 `43.153.143.37` 即可。不要和 etedrop 共用同一个本机应用端口。

## 3. 初始化服务器

登录服务器：

```bash
ssh root@你的服务器IP
```

更新系统并安装基础工具：

```bash
apt update
apt upgrade -y
apt install -y git curl nginx certbot python3-certbot-nginx
```

安装 Node.js 24：

```bash
curl -fsSL https://deb.nodesource.com/setup_24.x | bash -
apt install -y nodejs
node -v
npm -v
```

安装 PM2：

```bash
npm install -g pm2
```

## 4. 拉取项目

建议放在 `/var/www`：

```bash
mkdir -p /var/www
cd /var/www
git clone 你的仓库地址 dropmini
cd dropmini
npm install
```

如果不是用 git，也可以把项目上传到 `/var/www/dropmini`。

## 5. 配置环境变量

在服务器项目目录创建 `.env.local`：

```bash
nano .env.local
```

生产环境最少配置：

```env
NEXT_PUBLIC_SITE_URL=https://dropmini.com
NEXT_PUBLIC_SITE_NAME=DropMini
NEXT_PUBLIC_SITE_DESCRIPTION=DropMini is a global shop for collectible plush dolls, military tank models, aircraft miniatures, and display-ready hobby pieces.
NEXT_PUBLIC_CONTACT_EMAIL=hello@dropmini.com
# NEXT_PUBLIC_CONTACT_INSTAGRAM=@dropmini
# NEXT_PUBLIC_CONTACT_WHATSAPP=+1 000 000 0000
# NEXT_PUBLIC_CONTACT_TIKTOK=@dropmini
# NEXT_PUBLIC_CONTACT_TWITTER=@dropmini

# true 显示购物车蒙版并禁用结账；false 隐藏蒙版并允许填写/支付。
NEXT_PUBLIC_CHECKOUT_PAUSED=true

DEFAULT_ADMIN_EMAIL=admin@dropmini.com
DEFAULT_ADMIN_PASSWORD=请换成强密码
DEFAULT_ADMIN_NAME=DROP MINI ADMIN

SHIPPING_FLAT_RATE_USD=24.99

NEXT_PUBLIC_APPLE_PAY_ENABLED=false
APPLE_PAY_MERCHANT_ID=

PAYPAL_CLIENT_ID=
PAYPAL_SECRET=
PAYPAL_API_BASE_URL=https://api-m.paypal.com
# PAYPAL_WEBHOOK_ID=
```

注意：

- `NEXT_PUBLIC_SITE_URL` 必须是线上域名，否则 sitemap、分享链接、SEO 地址会不正确。
- 默认管理员只在数据库没有管理员时创建。已有管理员请在后台 `/admin/settings` 修改。
- 如果暂时不接在线支付，PayPal 可以留空。

## 6. 构建和启动

执行生产构建：

```bash
npm run build
```

用 PM2 启动：

```bash
PORT=40322 pm2 start npm --name dropmini -- start
pm2 save
pm2 startup
```

查看状态：

```bash
pm2 status
pm2 logs dropmini
```

生产环境建议监听 `40322` 端口，避免和服务器上的其他服务冲突。Nginx 只需要反向代理到 `127.0.0.1:40322`。

## 7. 配置 Nginx

首次部署时，`/etc/letsencrypt/live/dropmini.com/fullchain.pem` 还不存在，不能直接启用含 `listen 443 ssl` 的完整配置。先写入只用于 ACME 校验的 HTTP 配置，签好证书后再换成完整 HTTPS 配置。

创建站点配置：

```bash
nano /etc/nginx/sites-available/dropmini
```

首次部署先写入：

```nginx
server {
    listen 80;
    listen [::]:80;
    server_name dropmini.com www.dropmini.com;

    location /.well-known/acme-challenge/ {
        root /var/www/html;
    }

    location / {
        return 301 https://dropmini.com$request_uri;
    }
}
```

启用配置：

```bash
ln -sf /etc/nginx/sites-available/dropmini /etc/nginx/sites-enabled/dropmini
nginx -t
systemctl reload nginx
```

## 8. 配置 HTTPS

确认 DNS 已经解析到服务器后执行：

```bash
certbot certonly --webroot -w /var/www/html -d dropmini.com -d www.dropmini.com
```

如果域名使用 Cloudflare 代理橙云，证书签发失败时可以临时把 `dropmini.com` 和 `www.dropmini.com` 切到 DNS only，签完证书后再切回 Proxied。Cloudflare SSL/TLS 模式建议使用 Full 或 Full strict。

签完证书后，确认文件存在：

```bash
ls -la /etc/letsencrypt/live/dropmini.com/fullchain.pem
ls -la /etc/letsencrypt/live/dropmini.com/privkey.pem
```

如果服务器上已有其他站点也定义了同名 `map $http_upgrade $connection_upgrade`，完整配置里只能保留一份 `map`。例如 EteDrop 配置已包含该 `map` 时，DropMini 配置里的 `map` 段可以删除；如果没有任何站点定义这个 `map`，则必须保留。

然后把 `/etc/nginx/sites-available/dropmini` 替换为完整 HTTPS 配置：

```nginx
map $http_upgrade $connection_upgrade {
    default upgrade;
    ''      close;
}

upstream dropmini_next {
    server 127.0.0.1:40322;
    keepalive 16;
}

server {
    listen 80;
    listen [::]:80;
    server_name dropmini.com www.dropmini.com;

    location /.well-known/acme-challenge/ {
        root /var/www/html;
    }

    location / {
        return 301 https://dropmini.com$request_uri;
    }
}

server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name www.dropmini.com;

    ssl_certificate     /etc/letsencrypt/live/dropmini.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/dropmini.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    return 301 https://dropmini.com$request_uri;
}

server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name dropmini.com;

    ssl_certificate     /etc/letsencrypt/live/dropmini.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/dropmini.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    client_max_body_size 20M;

    location / {
        proxy_pass http://dropmini_next;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

若 `nginx -t` 报 `/etc/letsencrypt/options-ssl-nginx.conf` 或 `/etc/letsencrypt/ssl-dhparams.pem` 不存在，先补齐 Certbot 的 SSL 参数文件：

```bash
mkdir -p /etc/letsencrypt
cp /usr/lib/python3/dist-packages/certbot_nginx/_internal/tls_configs/options-ssl-nginx.conf \
  /etc/letsencrypt/options-ssl-nginx.conf
cp /usr/lib/python3/dist-packages/certbot/ssl-dhparams.pem \
  /etc/letsencrypt/ssl-dhparams.pem
```

如果路径不存在，先查找实际位置：

```bash
find /usr /snap -name options-ssl-nginx.conf -o -name ssl-dhparams.pem
```

最后重载 Nginx：

```bash
nginx -t
systemctl reload nginx
```

测试自动续期：

```bash
certbot renew --dry-run
```

## 9. 备份

生产数据主要在：

```text
data/yezi.db
public/uploads/
.env.local
```

建议每天备份一次：

```bash
mkdir -p /var/backups/dropmini
tar -czf /var/backups/dropmini/dropmini-$(date +%F).tar.gz \
  /var/www/dropmini/data \
  /var/www/dropmini/public/uploads \
  /var/www/dropmini/.env.local
```

建议再把 `/var/backups/dropmini` 同步到对象存储、另一台服务器或本地电脑。

每次更新代码：

```bash
cd /var/www/dropmini
git pull
npm install
npm run build
pm2 restart dropmini
```

如果修改了 `PORT` 或其他环境变量，使用：

```bash

```

更新后查看日志：

```bash
pm2 logs dropmini
```

## 13. 常见问题

### 502 Bad Gateway

检查应用是否启动：

```bash
pm2 status
pm2 logs dropmini
```

检查 Nginx：

```bash
nginx -t
systemctl status nginx
```

### 上传图片失败

检查目录权限：

```bash
mkdir -p /var/www/dropmini/public/uploads
chown -R root:root /var/www/dropmini/public/uploads
chmod -R 755 /var/www/dropmini/public/uploads
```

如果使用非 root 用户运行 PM2，请把目录 owner 改成运行应用的用户。

### 修改 `.env.local` 后没有生效

重新构建并重启：

```bash
npm run build
pm2 restart dropmini
```

### 管理员密码忘记

如果数据库里仍是默认管理员，可先改 `.env.local` 的 `DEFAULT_ADMIN_PASSWORD`，重启后用默认管理员邮箱登录。  
如果已经在后台改过密码，需要直接修改数据库或重新创建管理员。
